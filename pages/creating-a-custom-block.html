Minecraft Modding 1.20.1 - Creating a Custom Block

Introduction

In this section, we will walk through how to create a custom block in Minecraft for modding. Custom blocks can add interesting gameplay mechanics or simply make your mod more immersive. We will cover defining the block, registering it, adding a texture, and providing basic interactions like block breaking and placement.

Creating the Custom Block

1. Define the Block Class

Start by creating a new Java class that will define your custom block. The block class extends Block and includes the basic properties of the block, such as its material, hardness, and resistance to explosions.

package com.example.mymod.blocks;

import net.minecraft.block.Block;
import net.minecraft.block.BlockState;
import net.minecraft.block.Material;
import net.minecraft.block.ShapeContext;
import net.minecraft.util.math.BlockPos;
import net.minecraft.world.BlockView;

public class CustomBlock extends Block {
    public CustomBlock() {
        super(Settings.of(Material.STONE).strength(3.0f, 3.0f)); // Block material and strength
    }

    @Override
    public boolean isFullBlock(BlockState state) {
        return true; // Custom behavior for block shape
    }

    @Override
    public boolean canSustainPlant(BlockState state, BlockView world, BlockPos pos, net.minecraft.util.math.Direction direction, net.minecraft.world.gen.feature.ConfiguredFeature feature) {
        return false; // Custom behavior for plant interaction
    }
}

This basic custom block extends the Block class, sets its material to stone, and defines its hardness and resistance to explosions. You can customize this further by adding more functionality.

2. Register the Custom Block

Next, you need to register your custom block. Just like with custom items, Minecraft needs to be aware of the new block. Register it in your mod's main class.

package com.example.mymod;

import com.example.mymod.blocks.CustomBlock;
import net.minecraft.block.Block;
import net.minecraft.util.Identifier;
import net.minecraft.util.registry.Registry;
import net.fabricmc.api.ModInitializer;

public class MyMod implements ModInitializer {
    public static final Block CUSTOM_BLOCK = new CustomBlock();

    @Override
    public void onInitialize() {
        Registry.register(Registry.BLOCK, new Identifier("mymod", "custom_block"), CUSTOM_BLOCK);
    }
}

In this example, we register the custom block under the namespace "mymod" with the identifier "custom_block". Again, you can change these to reflect your mod and block names.

3. Add a Block Texture

Now, let’s add a texture for our custom block. Create a texture file and place it in the following directory: src/main/resources/assets/mymod/textures/block/. The texture should be named custom_block.png.

src/main/resources/assets/mymod/textures/block/custom_block.png

This will display the texture for your custom block in the game. You can design it as you like using any image editor.

4. Creating the Block Item

When a block is placed in the world, players can interact with it, but it also needs to be added to the player's inventory. Let’s create an item for our block so it can be picked up and placed by the player.

package com.example.mymod.items;

import net.minecraft.item.BlockItem;
import net.minecraft.item.Item;
import net.minecraft.item.ItemGroup;
import net.minecraft.util.Identifier;
import net.minecraft.util.registry.Registry;
import net.minecraft.block.Block;

public class CustomBlockItem extends BlockItem {
    public CustomBlockItem(Block block) {
        super(block, new Item.Settings().group(ItemGroup.BUILDING_BLOCKS)); // Adds to building blocks tab
    }
}

// Register the item
public class MyMod implements ModInitializer {
    public static final Block CUSTOM_BLOCK = new CustomBlock();
    public static final Item CUSTOM_BLOCK_ITEM = new CustomBlockItem(CUSTOM_BLOCK);

    @Override
    public void onInitialize() {
        Registry.register(Registry.ITEM, new Identifier("mymod", "custom_block_item"), CUSTOM_BLOCK_ITEM);
    }
}

This code creates a new BlockItem for our custom block and adds it to the "Building Blocks" inventory tab. Now players can pick up and place the custom block.

5. Testing the Custom Block

Once everything is set up, you can run your mod and test your custom block. To do this, use the following command to add it to your inventory:

/give @p mymod:custom_block_item 1

Place the custom block in the world and interact with it. You should see your custom block in the game with the texture and behavior you defined.
